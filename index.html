Mục 1 // 15'

Mindset — tư duy product, không phải execution

AI execute giùm phần nhanh & rẻ → thời gian dôi ra mình tái đầu tư vào skill ngoài code và thử nghiệm nhiều hơn.
  • Code là sawdust — AI execute nhanh & rẻ. Tài sản thật = prompt + product thinking.
  • Thời gian dôi ra → build skill ngoài code: đi research · audit app khác tự động · so sánh sản phẩm · tìm cải tiến.
  • Invent thứ mới cho sản phẩm — làm cái chưa ai làm; cái lặp lại để AI lo.
  • Thử nghiệm nhiều hơn. Test xong có thể không ship · không “point” — nhưng test 1 ý tưởng = learn rất nhiều, eureka. Đo bằng số idea test/tuần.
  • Tiêu chuẩn cao như fine dining — AI execute rẻ thì taste mới là khác biệt. 80% idea fail vì bar nâng lên, không phải cẩu thả. “fail predictably > succeed unpredictably.”
  • ⚠ Verify: AI hay tự nhận “xong” khi mới nửa vời → mình kiểm.

Mục 1 // mindset · học suốt đời

Learning — học cái mình chưa từng học

Thời AI, learn không chỉ là learn kỹ thuật. Kỹ thuật để AI lo phần lớn → lợi thế nằm ở thứ mình chưa bao giờ học.

Mục 2 // /insights · soi chính mình

Thói quen vận hành — đo chính mình

Mình chạy /insights soi ~20k prompt của chính mình. Số liệu nói thật hơn cảm giác.
  • 99% prompt bằng English — habit này mình giữ được.
  • ~30% prompt ≤ 4 từ · median 8 từ — gọn, tiết kiệm token.
  • clear · compact · resume dùng đều tay — context hygiene ổn.
  • Plan mode trước khi code · verify trước khi commit — chốt vòng.

Rút ra

  • ✓ Giữ: English-first · prompt gọn · clear context đều tay.
  • ⚠ Cảnh giác: AI hay nhận “xong” khi nửa vời → luôn tự verify (test/screenshot).
  • → Tận dụng: mình hay debug bằng screenshot → đẩy mạnh Claude-in-Chrome / computer use.

Mục 5 // ralph loops · talk AI Engineer

Ralph Loops — “dumb loops that ship”

Chris Parsons (Cherrypick). Luận điểm gốc: dumb loop > clever workflow — đừng vội multi-agent orchestration / planning graph (debug hàng tháng).
  • 1 loop đơn giản (while) xử lý từng ticket end-to-end → thắng cả đống tool chain rườm rà.
  • Feedback loop là trái tim: TDD (red→green) + synthetic loop để test ở local, không chờ data production.
  • Tự cải thiện: output tốt lên mỗi vòng mà không cần sửa prompt — loop tích lũy.

Validate + Exit

Sub-agent “critic” chấm mỗi vòng (= generator–evaluator) · điều kiện dừng rõ cho task mở · canh context rot · vẫn review & verify tay.

Tự động hóa thật

worker / morning loop chạy nền định kỳ · /loop command (≈ plugin ralph-loop) · gắn vào ticket system → mỗi vòng ship 1 ticket.

Mục 5 // cảnh báo · token là tiền thật

Loop ≠ chạy vô tận — kẻo cháy token

“Lúc nào cũng chạy loop” thực ra chỉ hợp với ai có token vô hạn. Loop lỏng để tự chạy mãi → đốt token cả tuần như chơi.

⚠ Bẫy đốt token

  • Loop vô hạn, không exit → chạy qua đêm vô tội vạ.
  • Mở nhiều loop lỏng cùng lúc → nhân chi phí token.
  • Để agent tự “chạy tiếp” không verify → đốt token vào AI slop.

Quy tắc loop của mình

  • Luôn có điều kiện exit + giới hạn vòng (max N rounds).
  • 1 mục tiêu rõ / loop — ít loop mà chặt hơn nhiều loop lỏng.
  • Verify mỗi vòng (generator–evaluator) → đạt là dừng ngay.
  • Cắt loop không hội tụ; clear context giữa phase.
Đo token/tuần. Loop là dao 2 lưỡi: chặt thì sắc, lỏng thì đốt tiền thật.
Mục 5 // cách mình prompt 1 loop

Prompt loop — chặt ngay từ đầu

  • Mục tiêu đo được — “0 test fail”, không phải “làm cho tốt”.
  • Input từng món — 1 ticket / 1 item mỗi vòng, không ôm cả đống.
  • Việc rõ mỗi vòng + bước verify (tự chấm trước khi qua vòng).
  • EXIT + MAX ROUNDS bắt buộc — hết vòng thì DỪNG & báo cáo.
  • NEVER list — chặn loop đi lạc (đụng file ngoài phạm vi…).
/ralph-loop# loop: fix failing tests
GOAL: all tests green
EACH ROUND:
  1 run `npm test`, take 1st fail
  2 fix root cause (don't edit the test)
  3 re-run + verify
EXIT WHEN: 0 failures
MAX ROUNDS: 8 → STOP + report
NEVER: touch unrelated files

Mục 6 // Nguyên tắc tổng quát (không chỉ UI)

5 nguyên tắc mình theo khi làm codebase cho agent

1 Consistency = constraint

1 cách làm 1 việc → agent ít sáng tác bậy. AI slop sinh ra ở chỗ cho phép 5 cách.

2 Self-runnable loop

Agent tự chạy + tự verify (dev/storybook/test/lint 1 lệnh) → tự sửa tới khi xanh.

3 Skill = quy trình · Agent = vai

Skill “add-ui-block” cho ai invoke đều cùng chất lượng; agent “ui-builder” chạy song song.

4 Living contract

Design system + skill = contract mà generator/evaluator chốt “done” trên đó — không cảm tính.

5 Preview = sân chơi evaluator

Mỗi component có preview cô lập → evaluator mở từng cái chấm, không cần dựng cả app.

Chứng minh

plan→ship không gõ code tay · clear context giữa phase · mình verify cái AI nhận “xong”.

FALLBACK → mở recording · hoặc review diff có sẵn

case: lấy 1 ticket mới live trên kênh #support (Slack)

Check bug → tạo task — đủ key, không cần browser

Kịch bản

Chộp 1 case mới trên #support → đọc thread + Crisp → query Firestore → root cause → brainstorm fix → tạo task.

DEMO3
super power: sub-agent development — chạy song song

Nhiều việc độc lập — cùng một lúc

Kịch bản

Giao 2–3 việc độc lập: rà i18n thiếu · tìm chỗ dùng useTheme · đọc changelog → chạy nền, báo về khi xong (push).

Chứng minh

Agent ≠ Skill: agent = vai/context riêng để parallel; skill = đóng gói quy trình. Main context vẫn gọn.

FALLBACK → chỉ chạy 2 agent · hoặc mô tả qua sơ đồ
