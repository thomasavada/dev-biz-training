Mục 1 // 15'

Mindset — tư duy product, không phải execution

  • Code là sawdust — mạt cưa rơi ra từ việc mô tả thứ mình muốn. Prompt + product thinking mới là tài sản.
  • Mình làm cái chưa làm, không làm lại cái đã làm. Hướng tới thứ mới/khó; cái lặp lại để AI lo.
  • Mình dám rewrite thay vì vá víu. Gen code rẻ → viết lại sạch nhiều khi nhanh & tốt hơn.
  • Mình giờ như nhạc trưởng / CEO công ty nhỏ: chỉ đạo nhiều agent, review & merge — không gõ tay.
  • “fail predictably > succeed unpredictably.” Vứt 80% thử nghiệm. Đo bằng số idea test/tuần, không phải dòng code.
  • ⚠ Context anxiety & AI slop: AI hay tự nhận “xong” khi mới nửa vời → mình phải verify.
Mục 1 // mindset · học suốt đời

Learning — học cái mình chưa từng học

Thời AI, learn không chỉ là learn kỹ thuật. Kỹ thuật để AI lo phần lớn → lợi thế nằm ở thứ mình chưa bao giờ học.
  • Học rộng ngoài code: business · ecom (sâu) · branding · marketing — hiểu sản phẩm & khách hàng, không chỉ cú pháp.
  • Dev = product thinker / CEO nhỏ (nối Mục 1) → quyết định “làm gì” quan trọng hơn “code thế nào”.
  • Kỹ thuật outsource cho AI được — hiểu business / khách hàng thì không. Đó mới là lợi thế.

Claude Local — setup “siêu năng lực”

.env.agent đầy đủ

Skills kết nối Crisp · Support · Ticket · Logging. Key theo level support → mình tự debug bug ở local hoặc qua OpenClaw.

Agent ≠ Skill

Agent = 1 context window riêng → chạy song song (viết rõ từng agent). Skill = đóng gói 1 quy trình lặp lại (viết khi muốn “đúng gói”).

  • Mình ưu tiên sub-agent development (super power) → tiết kiệm token.
  • Mình luôn plan mode trước → plan xong → clear → implement (tránh spam context).
  • Mình giữ đơn giản: Warp hoặc Zed là đủ; đọc luồng code ở mức cơ bản.
  • Checklist setup gửi trước buổi → ai cũng có môi trường chạy ngay.

Mục 5 // ralph loops · talk AI Engineer

Ralph Loops — “dumb loops that ship”

Chris Parsons (Cherrypick). Luận điểm gốc: dumb loop > clever workflow — đừng vội multi-agent orchestration / planning graph (debug hàng tháng).
  • 1 loop đơn giản (while) xử lý từng ticket end-to-end → thắng cả đống tool chain rườm rà.
  • Feedback loop là trái tim: TDD (red→green) + synthetic loop để test ở local, không chờ data production.
  • Tự cải thiện: output tốt lên mỗi vòng mà không cần sửa prompt — loop tích lũy.

Validate + Exit

Sub-agent “critic” chấm mỗi vòng (= generator–evaluator) · điều kiện dừng rõ cho task mở · canh context rot · vẫn review & verify tay.

Tự động hóa thật

worker / morning loop chạy nền định kỳ · /loop command (≈ plugin ralph-loop) · gắn vào avada-tickets → mỗi vòng ship 1 ticket Joy.

Mục 6 // 15' · case study sống

Agent-friendly codebase — web component v4

Cách mình thiết kế code để agent làm tốt. Case thật: task web component v4.
  • Mình xây design system đủ thành phần → “bảng màu + nốt nhạc” để agent ghép, không tự chế từ đầu mỗi lần.
  • Mình đóng gói thành skill (“thêm 1 UI block”) + agent (context window riêng chạy luồng đó).
  • Invoke flow → agent imagine thêm component mới liên tục mà vẫn consistent, vì bị ràng bởi hệ thống có sẵn, không drift.
  • Agent tự vite lên → tự xem → tự sửa UI → tự test lại = vòng generator–evaluator chạy ngay trên UI thật.
Mục 6 // Nguyên tắc tổng quát (không chỉ UI)

5 nguyên tắc mình theo khi làm codebase cho agent

1 Consistency = constraint

1 cách làm 1 việc → agent ít sáng tác bậy. AI slop sinh ra ở chỗ cho phép 5 cách.

2 Self-runnable loop

Agent tự chạy + tự verify (dev/storybook/test/lint 1 lệnh) → tự sửa tới khi xanh.

3 Skill = quy trình · Agent = vai

Skill “add-ui-block” cho ai invoke đều cùng chất lượng; agent “ui-builder” chạy song song.

4 Living contract

Design system + skill = contract mà generator/evaluator chốt “done” trên đó — không cảm tính.

5 Preview = sân chơi evaluator

Mỗi component có preview cô lập → evaluator mở từng cái chấm, không cần dựng cả app.

Software fundamentals // Matt Pocock

Code KHÔNG rẻ

  • Trào lưu “viết spec → AI gen → lỗi thì sửa spec, đừng nhìn code” = vibe coding đổi tên.
  • Chạy lại compiler/agent nhiều lần mà không nhìn code → code càng ngày càng rác (software entropy).
  • Mình vẫn đọc code, áp taste — AI không thay được trách nhiệm.
